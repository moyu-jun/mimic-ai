# Mimic 需求文档

> 本文档描述 Mimic 应用要实现的功能、用户行为约束与边界条件。
> 技术选型、模块划分、命令/事件签名等实现细节见 [DESIGN.md](./DESIGN.md)。
> 实施顺序、阶段验收和跟踪见 [TASKS.md](./TASKS.md)。

## 1. 项目概述

Mimic 是一款仅面向 Windows 的桌面按键/鼠标模拟工具，主要服务于游戏场景下的键盘按键和鼠标点击模拟。用户通过界面配置要模拟的按键序列、点击坐标以及全局热键；按下启动热键后由后台循环执行模拟动作，按下停止热键后退出循环。

应用本身负责配置管理、状态展示与全局热键监听；底层模拟动作通过第三方 Interception 驱动完成。

## 2. 适用平台与运行约束

- 仅支持 Windows，不考虑 macOS 与 Linux。
- 模拟功能依赖第三方 Interception 驱动；驱动未安装或安装后未重启时，模拟功能不可用，但配置界面必须能正常打开。
- 应用以无系统标题栏、固定大小、深色风格的窗口运行。
- 驱动安装与按键/鼠标模拟均需要管理员权限。应用采用**「启动时请求提权 + 拒绝降级」**策略：启动时主动弹出 UAC 提权请求，**用户同意**则以管理员权限正常运行，**用户拒绝**则以普通权限继续启动并展示配置界面，但驱动安装与模拟功能不可用，需要由用户在首页主动重新触发以管理员身份重启。
  - 启动时由后端运行时检测当前进程是否具有管理员权限：
    - 已提权 → 直接进入主界面（首页绿色「已授予」）。
    - 未提权 → 主动调用 `ShellExecuteW("runas")` 弹出 UAC：
      - 用户同意 → 提权后的新进程接管，当前进程立即退出，避免双开。
      - 用户拒绝（或 UAC 失败）→ 降级启动，首页显示橙色「管理员权限受限,部分功能不可用」 + 「以管理员身份重启」按钮。
  - **不在程序清单（manifest）中强制 `requireAdministrator`**，避免拒绝 UAC 时应用直接启动失败 — 由后端运行时主动调度 UAC,获得「拒绝即降级」的灵活性。
  - 驱动安装在未授权时由后端通过 `runas` 以管理员身份重启安装进程；模拟运行在未授权时拒绝启动并提示「需要管理员权限」。

## 3. 功能需求

### 3.1 应用窗口与外观

- 窗口尺寸固定为 `600 x 400`，禁止用户缩放、最大化或拖拽改变大小。
- 窗口不使用系统标题栏，由应用自行实现，标题栏自左到右包含：标题（应用图标 + 应用名 `Mimic`）、当前页面名（居中显示，如"首页""按键模拟"等）、最小化按钮、关闭按钮。
- 当前运行状态摘要放在底部状态栏（见 3.10），不在标题栏。
- 标题栏存在拖拽区域，必须支持鼠标拖动整个窗口。
- 窗口最外层圆角，配合透明背景实现现代化外观。
- 整体使用深色现代风格，参考 Windows 11 暗色主题：
  - 深灰背景、中灰容器、浅灰边框、系统蓝强调色，配合状态色（绿/黄/红）。
  - 元素具备圆角、柔和边框、微妙阴影。
- 交互反馈使用 120ms-220ms 的过渡动画，覆盖 hover、active、focus 与页面切换。

### 3.2 主界面结构

界面布局自上而下、自左而右固定为：

- 顶部：自定义标题栏，全局固定，不随页面切换。
- 中部主区域：左侧菜单栏 + 右侧页面内容区。
- 底部：状态栏，全局固定，显示当前运行状态。

左侧菜单包含以下入口：

- 首页
- 按键模拟
- 鼠标模拟
- 设置（固定置于左侧菜单底部）

### 3.3 应用页面

#### 3.3.1 首页

提供应用整体说明与快速进入入口。具体内容由设计文件细化，不影响其他页面行为。

#### 3.3.2 按键模拟页面

- 展示按键动作列表。
- 提供按键捕获输入框和"添加"按钮，用于新增按键动作。
- 列表每行依次为：多选框、键位信息、间隔时间（ms）、删除按钮（垃圾桶图标样式）。
- 每行支持勾选/取消勾选、修改间隔时间、删除该行。
- 间隔时间输入框仅接受正整数，禁止负号与非数字字符，且输入框不带加减步进按钮。
- 勾选状态与未勾选状态需要有明显的视觉差异（颜色、透明度、强调线、勾选框等），但**禁止使用文字删除线**。
- 列表需提供滚动条以容纳较多数据，并预留滚动条宽度空间，避免出现滚动条时列表宽度突变影响观感。
- 此页面属于"可触发模拟"页面，启动热键在该页面生效。

#### 3.3.3 鼠标模拟页面

- 展示鼠标点击动作列表。
- 每行包含 X 坐标、Y 坐标、间隔时间（ms）、"坐标拾取"按钮、删除按钮（垃圾桶图标样式）。
- 每行支持修改间隔时间、删除该行。
- X、Y 坐标不允许手动输入，仅可通过"坐标拾取"按钮修改。
- 间隔时间输入框仅接受正整数，禁止负号与非数字字符，且输入框不带加减步进按钮。
- 列表下方提供"添加"按钮，点击后追加一行空坐标数据。
- 列表需提供滚动条以容纳较多数据，并预留滚动条宽度空间，避免出现滚动条时列表宽度突变影响观感。
- 此页面属于"可触发模拟"页面，启动热键在该页面生效。

#### 3.3.4 设置页面

- 提供启动热键和停止热键的捕获输入框。输入框获得焦点后进入捕获状态，显示"请按下热键"提示并自动捕获按键回显；若未按下任何按键就点击别处导致失焦，则回显失焦前的原热键信息。
- 提供保存按钮。用户修改热键后需手动点击保存；保存时对比当前配置与已持久化配置，无变化则保持不变，有变化则重新注册热键并持久化到配置文件。
- 点击保存时：无变化不提示；有变化则对热键注册成功/失败、持久化成功/失败给出小文字提示。
- 启动热键和停止热键允许为同一个键位。
- 此页面**不是**可触发模拟页面，启动/停止热键在该页面不应触发模拟，但仍应正常拦截已注册的热键输入。

### 3.4 配置数据

应用需要管理并持久化以下配置：

- **按键动作集合**：每个动作包含动作 ID、是否勾选、按键显示名、按键 scan code、动作执行后等待的间隔毫秒数。
- **鼠标动作集合**：每个动作包含动作 ID、屏幕 X 坐标（可空）、屏幕 Y 坐标（可空）、间隔毫秒数。
- **全局热键配置**：包含启动热键和停止热键，每个热键包含按键显示名与 scan code。

当前版本**不需要**考虑组合键（如 Shift + S），所有按键以单键处理。如未来需要组合热键，再以独立需求补回。

### 3.5 配置持久化

- 配置文件名固定为 `mimic.ini`，与可执行文件位于同一目录。
- 启动时：
  - 文件存在 → 读取并解析。
  - 文件不存在 → 使用默认配置初始化并立即创建文件。
  - 文件损坏（无法解析） → 直接用默认配置覆盖原文件，不创建备份。
- 用户修改任何配置后需要持久化；启动/停止热键仅在点击保存按钮且数据与已持久化数据不一致时才写盘。
- 持久化时机：勾选、增删行等结构性变更立即写盘；间隔时间等数字输入在输入框失焦或提交时写盘，避免逐字符频繁写磁盘。

### 3.6 全局热键

- 默认启动热键和停止热键均为 `F12`。
- 启动热键和停止热键允许配置为同一个按键；此时由当前运行状态判断按下后应启动还是停止。
- 应用启动顺序必须是：先加载/初始化 `mimic.ini`，再使用加载到的配置注册全局热键。
- 设置页修改热键成功保存后必须实时更新全局热键注册。
- 应用窗口最小化或失焦后，全局热键仍然能够触发。
- 当前页面不属于"可触发模拟"页面（按键模拟、鼠标模拟）时，启动/停止热键不生效。
- 热键生效时机受当前运行状态约束：
  - 处于待机状态时，启动热键才能生效。
  - 处于运行状态时，停止热键才能生效。
  - 状态不匹配时按键被忽略。
- 处于运行状态时（即启动热键被触发后），**不允许**阻塞其他键盘键位的正常使用。
- 热键键位支持：字母键、数字键、F1-F12、Space、Tab、Esc、Left Shift、 Right Shift、Left Ctrl、Right Ctrl、Left Alt、Right Alt。与按键模拟模块的支持范围一致。

### 3.7 按键捕获

- 按键捕获输入框获得焦点后进入捕获状态。
- 捕获状态下监听键盘按下事件，**阻止默认输入行为**。
- 捕获结果包含按键的显示名与对应的 scan code。
- 当前版本不支持组合键，仅按单键处理。
- 必须支持游戏常用按键：字母键、数字键、F1-F12、Space、Tab、Esc、Left Shift、 Right Shift、Left Ctrl、Right Ctrl、Left Alt、Right Alt。
- 完成捕获后点击"添加"按钮可将该按键追加为按键模拟动作，默认未勾选、间隔为全局默认值(20ms为全局默认值)。

### 3.8 鼠标坐标拾取

- 在鼠标模拟页面对某一行点击"坐标拾取"按钮后进入拾取状态。
- 拾取过程中应用窗口必须**隐藏**，避免遮挡目标点击区域。
- 拾取下一次全局鼠标点击的屏幕坐标。
- 拾取完成后窗口恢复显示，并将坐标写入对应行，同时持久化。
- 第一版仅承诺单显示器、标准 DPI 缩放场景下可用；游戏窗口或全屏场景下若不可用，列入待确认事项。

### 3.9 模拟执行

- 启动热键触发后，根据当前页面执行对应类型的模拟：
  - **按键模拟页面**：取所有勾选的按键动作，按列表顺序执行 key down + key up；每个动作完成后等待该行配置的间隔时间；列表执行完成后从头循环。
  - **鼠标模拟页面**：取所有 X、Y 都有效（排除空数据）的鼠标动作，按列表顺序移动鼠标并执行左键点击；每个动作完成后等待该行配置的间隔时间；列表执行完成后从头循环。
- 鼠标模拟第一版仅支持左键。
- 模拟运行循环直到停止热键被触发。
- 模拟运行期间界面进入锁定状态：
  - 中部主区域（左侧菜单 + 右侧内容）显示一层半透明灰色蒙版。
  - 蒙版**不覆盖**顶部标题栏和底部状态栏。
  - 蒙版上**不显示**任何文字、按钮、图标或提示信息。
  - 禁止用户切换菜单、修改按键模拟、鼠标模拟和设置数据。
  - 当前运行状态仅在底部状态栏显示。
- 模拟运行期间，后端必须拒绝可能改变运行上下文的命令（保存配置、更新热键、坐标拾取、切换页面等），不能仅依赖前端蒙版作为防线。
- 停止热键触发并确认运行循环已退出后，界面恢复可操作。

### 3.10 运行状态

应用需要在状态栏可视化以下运行状态：

| 状态 | 含义 |
|------|------|
| `Idle` | 待机 |
| `ReadyKeyboard` | 当前处于按键模拟页面，可启动按键模拟 |
| `ReadyMouse` | 当前处于鼠标模拟页面，可启动鼠标模拟 |
| `RunningKeyboard` | 按键模拟运行中 |
| `RunningMouse` | 鼠标模拟运行中 |
| `PickingMouse` | 正在拾取鼠标坐标 |
| `Error` | 驱动或配置错误 |

### 3.11 Interception 驱动管理

- 应用启动时检测 Interception 驱动是否已安装。
- 驱动**未安装**时：
  - 应用必须能正常打开并展示配置界面，但模拟功能不可用。
  - 引导用户进行 Interception 驱动安装。
  - 安装完成后弹窗提示用户必须重启电脑。
  - 重启之前模拟功能保持不可用，状态栏给出"驱动待重启"或"驱动不可用"的提示。
- 驱动安装文件采用 exe 同级目录外置方案，目录为 `<exe_dir>/drivers/interception/`，便于用户和杀毒软件识别文件来源，也便于手动替换版本。
- 驱动检测或安装失败时，必须记录错误日志并在界面给出明确提示。
- 驱动检测可在普通权限下执行（仅读注册表与文件系统）；驱动安装与模拟运行需要管理员权限，未授权时按第 2 节降级启动策略处理（界面可用、相关功能拒绝执行并提示）。

### 3.12 日志

- 后端必须接入日志系统，方便排查配置文件、热键注册、驱动检测/安装、模拟执行等问题。
- 开发环境记录 `info` 及以上级别。
- 生产 release 包仅记录 `error` 级别。
- 日志至少需要覆盖：
  - 应用启动信息与配置文件路径。
  - `mimic.ini` 创建、读取、解析失败、覆盖默认配置等事件。
  - 全局热键注册、注销、更新结果。
  - Interception 驱动检测和安装结果。
  - 模拟任务启动、忽略重复启动、停止、异常退出。
  - 鼠标坐标拾取开始、完成、失败。

## 4. 默认配置

应用首次启动或配置损坏时，必须使用以下默认配置：

- **按键动作列表**：包含一项默认按键 `F`，scan code 33，间隔 20ms，处于已勾选状态。
- **鼠标动作列表**：包含一项默认鼠标动作，X 与 Y 均为空，间隔 20ms。
- **全局热键**：启动热键与停止热键均为 `F12`（scan code 88）。
- **全局默认间隔时间常量**：`20ms`。

## 5. 约束与风险

- 全局热键注册与 Interception scan code 不在同一抽象层，必须维护可靠的按键映射，避免热键和模拟之间的语义错位。
- Interception 驱动依赖外部系统安装，驱动缺失时不能影响应用本体启动。
- 游戏与反作弊环境可能拦截或禁止模拟输入，应用不能承诺所有游戏可用。
- 自定义透明圆角窗口在 Windows + Tauri + WebView2 下需要实机验证，CSS 圆角并不等同于系统窗口圆角。
- INI 文件不擅长承载复杂列表，列表字段需采用 JSON 字符串落地的方式，否则后期扩展和兼容成本会大幅上升。
- `600 x 400` 窗口空间紧张，所有页面布局必须紧凑，避免大量说明文字和大尺寸卡片。

## 6. 待确认事项

1. Interception 驱动外置目录下具体包含哪些文件、对应的安装命令是什么 — 待文件齐备后确认。
2. 鼠标坐标拾取第一版方案在游戏窗口或全屏场景下是否可用 — 如不可用则需要替换底层实现，但保持上层接口与事件协议不变。
