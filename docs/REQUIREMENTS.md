## 需求变更记录

> 本节记录项目需求的正式变更，每次变更分配唯一编号。

### REQ-CHANGE-001：热键支持范围扩展至修饰键

**变更时间**：2026-06-08  
**变更类型**：功能增强  
**优先级**：高

#### 原需求

热键支持范围（需求 3.6）：字母键、数字键、F1-F12、Space、Tab、Esc、Left Shift、Right Shift、Left Ctrl、Right Ctrl、Left Alt、Right Alt。

阶段 12 实现采用 `tauri-plugin-global-shortcut`（Windows RegisterHotKey API）时，受 API 限制，**不支持**修饰键（Ctrl/Alt/Shift）作为**独立**热键，仅支持修饰键作为组合键的一部分（如 Ctrl+A）。当时前端 keyMap.ts 与后端映射表中移除了修饰键映射，临时将可选热键限制为字母/数字/F1-F12/Space/Tab/Esc。

#### 新需求

用户需求必须支持 Ctrl/Alt/Shift（左右）作为**独立热键**，用于特定游戏场景的一键模拟启动。新需求将支持范围扩展为：
- 原支持项：字母键、数字键、F1-F12、Space、Tab、Esc
- 新增项：Left Ctrl、Right Ctrl、Left Alt、Right Alt、Left Shift、Right Shift

#### 变更原因

1. **用户反馈**：阶段 12 后用户测试反馈，某些游戏场景需要单个修饰键作为启动/停止热键。
2. **技术方案限制**：RegisterHotKey API 不支持纯修饰键作为热键（API 约束，至少需要一个非修饰键）。
3. **架构统一契机**：阶段 13 引入 Interception 驱动作为按键模拟的底层时，同步改为使用 Interception 进行热键监听。Interception 在内核层拦截所有按键事件，天然支持任何单键（包括修饰键）的识别，无此限制。

#### 影响评估

**前端影响**：
- [src/lib/keyMap.ts](../src/lib/keyMap.ts)：恢复 ShiftLeft/ShiftRight/ControlLeft/ControlRight/AltLeft/AltRight 的映射表条目。
- [src/pages/SettingsPage.vue](../src/pages/SettingsPage.vue)：无改动（已支持任意白名单键的捕获）。
- 用户在设置页可选择修饰键作为热键，前端行为不变。

**后端影响**：
- [src-tauri/src/hotkeys.rs](../src-tauri/src/hotkeys.rs)：移除对修饰键映射的临时限制（阶段 12 修复）。
- [src-tauri/src/hotkeys_interception.rs](../src-tauri/src/hotkeys_interception.rs)：新建于阶段 13，使用 Interception 驱动直接监听所有按键，包括修饰键。
- 驱动依赖关系：热键注册与模拟运行共用同一 Interception context，架构统一。
- 降级策略：驱动未安装时热键完全不可用（不回退到 RegisterHotKey），状态栏明确提示「驱动待安装」。

**相关文档变更**：
- [REQUIREMENTS.md](./REQUIREMENTS.md) 3.6 节：支持范围明确列出修饰键。
- [DESIGN.md](./DESIGN.md) 8.3 节：热键实现方案演进，记录从 global-shortcut 切换至 Interception 的动机与实现细节。
- [TASKS.md](./TASKS.md) 阶段 12/13：阶段 12 移除修饰键支持的临时措施、阶段 13 恢复支持的实现。

#### 验证结果

- **TypeScript 编译**：`npm run build` 成功，无类型错误。

- **数据格式**：配置文件 mimic.ini 格式不变，持久化正确。

## 1. 项目概述

Mimic 是一款仅面向 Windows 的桌面按键/鼠标模拟工具，主要服务于游戏场景下的键盘按键和鼠标点击模拟。用户通过界面配置要模拟的按键序列、点击坐标以及全局热键；按下启动热键后由后台循环执行模拟动作，按下停止热键后退出循环。同时提供热键提示音和自定义提示音录制功能。

应用本身负责配置管理、状态展示与全局热键监听；底层模拟动作通过第三方 Interception 驱动完成。

## 2. 适用平台与运行约束

- 仅支持 Windows，不考虑 macOS 与 Linux。
- 模拟功能依赖第三方 Interception 驱动；驱动未安装或安装后未重启时，模拟功能不可用，但配置界面必须能正常打开。
- 应用以无系统标题栏、固定大小、深色风格的窗口运行。
- 驱动安装需要管理员权限；驱动一旦安装并加载，按键/鼠标模拟在普通权限下即可运行。应用采用**「按需提权」**策略：启动时不主动请求 UAC，仅当用户点击「安装驱动」时由后端拒绝并引导提权。
  - 启动时由后端运行时检测当前进程是否具有管理员权限，仅作首页状态展示，不主动调度 UAC：
    - 已提权 → 首页绿色「管理员权限已授予」。
    - 未提权 → 首页橙色「管理员权限受限，安装驱动需提权」+ 「以管理员身份重启」按钮（用户主动点击才触发 UAC）。
  - **不在程序清单（manifest）中强制 `requireAdministrator`**，应用普通权限即可启动。
  - 驱动安装命令在未授权时直接返回 `permission_denied`，前端引导用户点击首页「以管理员身份重启」。重启后再点「安装驱动」可正常触发 UAC。
  - 安装完成并重启系统加载驱动后，再次启动应用即使是普通权限，热键监听与模拟均可正常工作。

## 3. 功能需求

### 3.1 应用窗口与外观

- 窗口尺寸固定为 `600 x 400`，禁止用户缩放、最大化或拖拽改变大小。
- 窗口不使用系统标题栏，由应用自行实现，标题栏自左到右包含：标题（应用图标 + 应用名 `Mimic`）、当前页面名（居中显示，如"首页""按键模拟"等）、最小化按钮、关闭按钮。
- 当前运行状态摘要放在底部状态栏（见 3.10），不在标题栏。
- 标题栏存在拖拽区域，必须支持鼠标拖动整个窗口。
- 窗口最外层圆角，配合透明背景实现现代化外观。
- 整体使用深色现代风格，参考 Windows 11 暗色主题：
  - 深灰背景、中灰容器、浅灰边框、系统蓝强调色，配合状态色（绿/黄/红）。
  - 元素具备圆角、柔和边框、微妙阴影。
- 交互反馈使用 120ms-220ms 的过渡动画，覆盖 hover、active、focus 与页面切换。

### 3.2 主界面结构

界面布局自上而下、自左而右固定为：

- 顶部：自定义标题栏，全局固定，不随页面切换。
- 中部主区域：左侧菜单栏 + 右侧页面内容区。
- 底部：状态栏，全局固定，显示当前运行状态。

左侧菜单包含以下入口：

- 首页
- 按键模拟
- 鼠标模拟
- 设置（固定置于左侧菜单底部）

### 3.3 应用页面

#### 3.3.1 首页

展示应用整体状态概览，包含：
- 应用名称、版本号、简要说明。
- 管理员权限状态（绿色已授权 / 橙色受限）及提权操作按钮。
- Interception 驱动状态卡片（四态：未安装 / 已安装待重启 / 已就绪 / 错误），含安装驱动/卸载驱动/重启电脑操作按钮。
- 当前热键配置概览（启动热键 / 停止热键）。

#### 3.3.2 按键模拟页面

- 展示按键动作列表。
- 提供按键捕获输入框和"添加"按钮，用于新增按键动作。
- 列表每行依次为：多选框、键位信息、间隔时间（ms）、删除按钮（垃圾桶图标样式）。
- 每行支持勾选/取消勾选、修改间隔时间、删除该行。
- 间隔时间输入框仅接受正整数，禁止负号与非数字字符，且输入框不带加减步进按钮；输入框失焦或按 Enter 时提交，中间输入过程允许暂时为空。
- 勾选状态与未勾选状态需要有明显的视觉差异（颜色、透明度、强调线、勾选框等），但**禁止使用文字删除线**。
- 列表需提供滚动条以容纳较多数据，并预留滚动条宽度空间，避免出现滚动条时列表宽度突变影响观感。
- 尝试添加已存在 scanCode 的按键时，显示 2 秒自动消失的橙色提示，不添加重复行。
- 此页面属于"可触发模拟"页面，启动热键在该页面生效。

#### 3.3.3 鼠标模拟页面

- 展示鼠标点击动作列表（表格形式，表头固定）。
- 每行包含 X 坐标、Y 坐标、间隔时间（ms）、"坐标拾取"按钮、删除按钮（垃圾桶图标样式）。
- 每行支持修改间隔时间、删除该行。
- X、Y 坐标不允许手动输入，仅可通过"坐标拾取"按钮修改；坐标未设置时显示「—」占位符。
- 间隔时间输入框仅接受正整数，禁止负号与非数字字符，且输入框不带加减步进按钮；输入框失焦或按 Enter 时提交，中间输入过程允许暂时为空。
- 列表下方提供"添加"按钮，点击后追加一行空坐标数据。
- 列表需提供滚动条以容纳较多数据，并预留滚动条宽度空间，避免出现滚动条时列表宽度突变影响观感。
- 此页面属于"可触发模拟"页面，启动热键在该页面生效。

#### 3.3.4 设置页面

- 提供「全局热键」区块：启动热键和停止热键的捕获输入框，以及保存按钮。
  - 输入框获得焦点后进入捕获状态，显示"请按下热键"提示并自动捕获按键回显；若未按下任何按键就点击别处导致失焦，则回显失焦前的原热键信息。
  - 保存按钮仅与热键持久化相关。用户修改热键后需手动点击保存；保存时对比当前配置与已持久化配置，无变化则静默，有变化则重新注册热键并持久化到配置文件，显示操作结果小文字提示（2 秒后自动消失）。
  - 启动热键和停止热键允许为同一个键位。
- 提供「提示音」区块（见 3.14）。
- 此页面**不是**可触发模拟页面，启动/停止热键在该页面不应触发模拟，但仍应正常拦截已注册的热键输入。

### 3.4 配置数据

应用需要管理并持久化以下配置：

- **按键动作集合**：每个动作包含动作 ID、是否勾选、按键显示名、按键 scan code、动作执行后等待的间隔毫秒数。
- **鼠标动作集合**：每个动作包含动作 ID、屏幕 X 坐标（可空）、屏幕 Y 坐标（可空）、间隔毫秒数。
- **全局热键配置**：包含启动热键和停止热键，每个热键包含按键显示名与 scan code。

当前版本**不需要**考虑组合键（如 Shift + S），所有按键以单键处理。如未来需要组合热键，再以独立需求补回。

### 3.5 配置持久化

- 配置文件名固定为 `mimic.ini`，**与可执行文件位于同一目录**（exe 同级目录），便于用户直接访问和随程序打包分发。
- 启动时：
  - 文件存在 → 读取并解析。
  - 文件不存在 → 使用默认配置初始化并立即创建文件。
  - 文件损坏（无法解析） → 直接用默认配置覆盖原文件，不创建备份。
- 用户修改任何配置后需要持久化；启动/停止热键仅在点击保存按钮且数据与已持久化数据不一致时才写盘。
- 持久化时机：勾选、增删行等结构性变更立即写盘；间隔时间等数字输入在输入框失焦或提交时写盘，避免逐字符频繁写磁盘。
- 模拟运行期间后端拒绝写盘（返回 "busy" 错误），前端静默忽略该错误，不影响用户操作。

### 3.6 全局热键

- 默认启动热键和停止热键均为 `F12`。
- 启动热键和停止热键允许配置为同一个按键；此时由当前运行状态判断按下后应启动还是停止。
- 应用启动顺序必须是：先加载/初始化 `mimic.ini`，再使用加载到的配置注册全局热键。
- 设置页修改热键成功保存后必须实时更新全局热键注册。
- 应用窗口最小化或失焦后，全局热键仍然能够触发。
- 当前页面不属于"可触发模拟"页面（按键模拟、鼠标模拟）时，启动/停止热键不生效。
- 热键生效时机受当前运行状态约束：
  - 处于待机（`Idle`/`Ready*`）状态时，启动热键才能生效。
  - 处于运行状态时，停止热键才能生效。
  - 状态不匹配时按键被忽略（透传，不影响系统正常使用）。
- 处于运行状态时（即启动热键被触发后），**不允许**阻塞其他键盘键位的正常使用。
- 热键键位支持：字母键、数字键、F1-F12、Space、Tab、Esc、Left Shift、Right Shift、Left Ctrl、Right Ctrl、Left Alt、Right Alt。与按键模拟模块的支持范围一致。

### 3.7 按键捕获

- 按键捕获输入框获得焦点后进入捕获状态。
- 捕获状态下监听键盘按下事件，**阻止默认输入行为**。
- 捕获结果包含按键的显示名与对应的 scan code。
- 当前版本不支持组合键，仅按单键处理。
- 必须支持游戏常用按键：字母键、数字键、F1-F12、Space、Tab、Esc、Left Shift、Right Shift、Left Ctrl、Right Ctrl、Left Alt、Right Alt。
- 白名单以外的按键静默忽略（继续等待下次按键，不显示错误）。
- 完成捕获后点击"添加"按钮可将该按键追加为按键模拟动作，默认未勾选、间隔为全局默认值（20ms）。

### 3.8 鼠标坐标拾取

- 在鼠标模拟页面对某一行点击"坐标拾取"按钮后进入拾取状态（`PickingMouse`）。
- 拾取过程中应用窗口必须**隐藏**，避免遮挡目标点击区域。
- 底层使用 Interception 内核驱动监听鼠标左键按下事件，支持在独占全屏游戏内拾取（用户态 hook 方案不支持全屏游戏）。
- 命中左键时透传点击事件（目标窗口仍能正常响应点击），同时通过 `GetCursorPos` 读取系统光标位置作为坐标。
- 拾取完成后窗口恢复显示，并将坐标写入对应行，同时持久化。
- 拾取过程**无取消机制**，用户只能通过左键点击完成拾取。
- 已知限制：部分独占全屏游戏锁定或隐藏系统光标时，`GetCursorPos` 可能返回非预期坐标。
- 第一版仅支持单显示器、标准 DPI 缩放场景；多显示器/高 DPI 坐标偏移为已知限制。

### 3.9 模拟执行

- 启动热键触发后，根据当前页面执行对应类型的模拟：
  - **按键模拟页面**：取所有勾选的按键动作，按列表顺序执行 key down + key up；每个动作完成后等待该行配置的间隔时间；列表执行完成后从头循环。若无勾选动作，热键被忽略。
  - **鼠标模拟页面**：取所有 X、Y 都有效（排除空数据）的鼠标动作，按列表顺序移动鼠标并执行左键点击；每个动作完成后等待该行配置的间隔时间；列表执行完成后从头循环。若无有效坐标，热键被忽略（不进入运行状态、不显示蒙版）。
- 鼠标模拟第一版仅支持左键。
- 模拟运行循环直到停止热键被触发。
- 模拟运行期间界面进入锁定状态：
  - 中部主区域（左侧菜单 + 右侧内容）显示一层半透明灰色蒙版。
  - 蒙版**不覆盖**顶部标题栏和底部状态栏。
  - 蒙版上**不显示**任何文字、按钮、图标或提示信息。
  - 禁止用户切换菜单、修改按键模拟、鼠标模拟和设置数据。
  - 当前运行状态仅在底部状态栏显示。
- 模拟运行期间，后端必须拒绝可能改变运行上下文的命令（保存配置、更新热键、坐标拾取、切换页面等），不能仅依赖前端蒙版作为防线。
- 停止热键触发并确认运行循环已退出后，界面恢复可操作。

### 3.10 运行状态

应用需要在状态栏可视化以下运行状态：

| 状态 | 含义 |
|------|------|
| `Idle` | 待机（首页或设置页） |
| `ReadyKeyboard` | 当前处于按键模拟页面，可启动按键模拟 |
| `ReadyMouse` | 当前处于鼠标模拟页面，可启动鼠标模拟 |
| `RunningKeyboard` | 按键模拟运行中 |
| `RunningMouse` | 鼠标模拟运行中 |
| `PickingMouse` | 正在拾取鼠标坐标 |
| `Recording` | 正在录制提示音 |
| `Error` | 驱动或配置错误 |

状态切换规则：
- 切换到按键/鼠标模拟页面时，状态自动切换到对应的 `Ready*` 状态（页面间可直接切换，无需经过 `Idle`）。
- 切换到首页或设置页时，状态切换回 `Idle`。
- 运行期间（`Running*` / `PickingMouse` / `Recording`）禁止切换页面（后端命令守卫拒绝）。

### 3.11 Interception 驱动管理

- 应用启动时检测 Interception 驱动是否已安装并加载。
- 驱动状态分四种：
  - `NotInstalled`：注册表无服务项，需安装。
  - `InstalledNeedReboot`：服务项存在但驱动未加载（已安装，需重启系统）。
  - `Ready`：驱动已加载，`create_context()` 可成功返回上下文，模拟功能可用。
  - `Error`：检测或初始化出现异常。
- 驱动**未安装**时：
  - 应用必须能正常打开并展示配置界面，但模拟功能不可用。
  - 引导用户进行 Interception 驱动安装。
  - 点击「安装驱动」时先判断管理员权限；未授权直接提示「以管理员身份重启」、不执行安装；管理员权限下才调用安装器。
  - 安装与卸载行为对齐：以安装命令成功返回作为「已安装待重启」信号（不依赖 `check_driver_status`，因驱动需重启才加载、检测在重启前不可靠）。安装成功后驱动卡片进入「驱动已安装，需重启电脑」展示，按钮切换为「重启电脑」，并以绿色提示文案引导用户重启。
  - 重启之前模拟功能保持不可用。
- 驱动安装文件采用 exe 同级目录外置方案，目录为 `<exe_dir>/drivers/interception/`，便于用户和杀毒软件识别文件来源，也便于手动替换版本。
- 驱动**已安装成功**（`Ready` 或 `InstalledNeedReboot`）时，首页驱动卡片在原安装按钮位置提供**「卸载驱动」**入口（红色警告色），用于移除驱动：
  - 卸载为高风险操作，采用**文字二次确认**防误触：点击「卸载驱动」后展开内联确认区，用户须在输入框中准确输入「卸载驱动」四字，再点「确认卸载」方可执行；输入不符则按钮禁用且给出提示。
  - **权限判断前置于确认框**：点击「卸载驱动」时先判断管理员权限；未授权直接提示「以管理员身份重启」，**不展开**输入确认区；仅在管理员权限下才出现文字输入确认框。
  - 卸载与安装对称，需管理员权限；未授权返回 `permission_denied`，处理方式同安装（首页提示 + 「以管理员身份重启」）。
  - 卸载执行后必须给出成功反馈：注册表服务项虽已移除，但驱动仍驻留内核，须重启系统才彻底生效。因此卸载成功后驱动卡片进入「驱动已卸载，需重启电脑」展示，卸载按钮**切换为「重启电脑」**，并以提示文案引导用户重启。
  - `InstalledNeedReboot` 状态下「重启电脑」与「卸载驱动」按钮并排展示。
- 驱动检测或安装失败时，必须记录错误日志并在界面给出明确提示。
- 驱动检测可在普通权限下执行（仅读注册表与文件系统）；**驱动安装与卸载**需要管理员权限，未授权时按第 2 节按需提权策略处理（首页提示 + 「以管理员身份重启」按钮）；模拟运行不需要管理员权限。

### 3.12 日志

- 后端必须接入日志系统，方便排查配置文件、热键注册、驱动检测/安装、模拟执行等问题。
- 开发环境记录 `info` 及以上级别。
- 生产 release 包仅记录 `error` 级别。
- 日志至少需要覆盖：
  - 应用启动信息与配置加载结果。
  - 管理员权限检测结果（`elevated` / `limited`）。
  - 驱动检测与安装操作。
  - 热键注册/注销操作及结果。
  - 模拟循环启动/停止事件。
  - 配置读写操作及错误。

### 3.13 热键提示音

- 启动热键**生效**时（模拟循环真正启动，进入 `Running*` 状态）播放开启提示音。
- 停止热键**生效**时（模拟循环真正停止，`Running*` → 待机状态）播放关闭提示音。
- 被忽略的热键按下（如运行中再按启动键、待机中按停止键）不播放提示音。
- 声音文件为 exe **同级目录**下的 `按键开启.wav`（启动）和 `按键关闭.wav`（停止），便于用户直接替换自定义音效。
- 文件不存在时静默跳过，不报错、不影响模拟功能。
- 若上一个提示音尚未播完就触发新提示音，新声音打断旧声音立即播放（后触发优先）。
- 提示音异步播放，不阻塞热键响应。

### 3.14 提示音录制

- 在设置页提供「提示音」区块，支持录制并替换 `按键开启.wav` 和 `按键关闭.wav`。
- 录制目标分两类：「启动提示音」对应 `按键开启.wav`，「停止提示音」对应 `按键关闭.wav`，互相独立。
- **录制设备**：系统默认麦克风输入，不提供设备选择 UI。
- **最大录制时长**：5 秒，到时自动停止录制。
- **录制流程**：
  1. 点击目标提示音的「录制」按钮，展开统一录制面板。
  2. 面板展开后并**不立即开始采集**，需用户主动点击「开始录制」才开始。
  3. 录制进行中显示实时波形动画。
  4. 用户点击「结束录制」或到达 5 秒上限时，录制停止，面板切换为静态波形（剪裁态）。
  5. 支持重录：在剪裁态点击「开始录制」可丢弃当前录音重新开始。
- **剪裁**：录制完成后展示全长静态波形，用户可拖动起始/结束标记选取保留范围（最小选区 100ms）。
- **试听**：点击「试听」按钮播放选区内容（Web Audio 本地播放，不写盘），可用进度线观察播放位置。
- **保存**：点击「保存」将选区内容裁剪后写盘替换原文件；写盘前自动释放 PlaySoundW 的文件句柄（`SND_PURGE`），写盘使用 `.tmp` 临时文件 + `fs::rename` 原子替换，失败时不破坏旧文件。
- **取消**：点击「取消」丢弃录音缓冲，面板关闭，不修改任何文件。
- **状态隔离**：
  - 录制期间状态栏显示「正在录制提示音」（`Recording` 状态）。
  - 录制期间运行态守卫拒绝其他写操作（模拟运行、坐标拾取等）。
  - 反之，模拟运行/坐标拾取期间也拒绝开始录制。
  - 离开设置页时自动取消进行中的录制。
- 面板中「试听已有提示音」按钮可在非录制状态下随时试听已保存的声音文件。
- 若麦克风不可用或初始化失败，通过事件通知前端显示错误提示。
